Can a Service Mesh Handle Authorization?

Yes, service meshes are well-equipped to handle authorization (often abbreviated as "authz"), particularly for service-to-service (east-west) traffic in microservices architectures. They enforce access control policies at the network layer, decoupling it from your application code (e.g., .NET services). This is done transparently via sidecar proxies (like Envoy), which inspect requests and apply rules before forwarding. Authorization in meshes typically builds on authentication (e.g., mTLS for identity) to decide what a service can access.

Meshes focus on fine-grained, declarative policies rather than coarse app-level checks, making them ideal for zero-trust environments. Popular implementations like Istio and Linkerd provide native support, while others (e.g., Consul) extend via plugins.

How It Works

  1. Policy Definition: Write YAML configs for rules based on attributes like source/destination identities, HTTP methods, paths, or JWT claims.
  2. Enforcement: The proxy evaluates policies (ALLOW/DENY/CUSTOM) on every request. CUSTOM can delegate to external systems (e.g., OPA for complex logic).
  3. Integration: Pairs with authn (e.g., Istio's mTLS) for full identity-based authz.

Benefits for Reliability & Security

  • Zero-Trust Enforcement: Automatically secures all traffic, reducing breach surfaces (e.g., deny-by-default).
  • Centralized Management: Update policies cluster-wide without redeploys, aiding compliance.
  • Observability: Logs denied requests for auditing, integrating with tools like Prometheus.
  • Decoupling: Offloads authz from services, simplifying .NET code (no more custom middleware for inter-service checks).

Limitations

  • Not for End-User Authz: Better for ingress gateways (e.g., pair with Envoy or Azure API Mgmt for north-south traffic).
  • Complexity: Steep learning curve for advanced policies; overkill for monoliths.
  • Performance: Adds ~1-2ms latency per hop, but tunable.

In a Tech Lead interview, highlight: "Service meshes excel at infra-level authz, but for app-specific logic, combine with libraries like IdentityServer in .NET." Recent evolutions (e.g., Ambient Mesh in 2024) make it even more efficient.
