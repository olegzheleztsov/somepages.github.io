Supported Tracing Frameworks in Istio

Istio provides robust support for distributed tracing to gain visibility into service interactions, leveraging Envoy proxies to collect and export trace data. As of the latest documentation (October 2025), Istio supports several popular tracing frameworks and backends through configurable integrations. These allow you to send traces from the mesh to external collectors without modifying application code. Configuration is typically done via Istio's Telemetry API, MeshConfig, or pod annotations, with options for sampling rates and formats.

Here's a summary of the supported frameworks:

Tracing Framework Description & Key Features Configuration Notes
OpenTelemetry Istio's preferred modern standard for traces, metrics, and logs. Supports exporting to any OTEL-compatible backend (e.g., Jaeger, Zipkin via OTEL collectors). Use the Telemetry API to enable OTEL export; proxies send traces in OTEL format. Ideal for new setups; see dedicated OTEL guide for proxy config.
Jaeger Open-source end-to-end distributed tracing system with UI for visualization. Handles sampling and storage. Configure proxies to send to Jaeger's collector endpoint (e.g., via UDP/TCP). Supports all-sampling or probabilistic modes; integrates with Elasticsearch/Cassandra backends.
Zipkin Lightweight, open-source tracing system focused on HTTP/JSON protocols. Simple setup with web UI. Proxies export spans to Zipkin's collector (HTTP POST). Best for smaller-scale meshes; configure via meshConfig.defaultConfig.tracing.zipkin.address.
Apache SkyWalking APM system with strong support for non-Java languages and plugins. Includes topology analysis. Send traces to SkyWalking's OAP server; supports gRPC/HTTP protocols. Useful for polyglot environments like .NET + Java services.

These integrations cover most use cases, with OpenTelemetry emerging as the go-to for future-proofing (e.g., vendor-agnostic exports).
