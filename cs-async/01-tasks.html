In C#, a Task represents an asynchronous operation, part of the Task Parallel Library (TPL) introduced in .NET Framework 4.0. It enables asynchronous programming, allowing operations to run concurrently without blocking the main thread.

  • Definition: A Task is an object that encapsulates a piece of work that can execute asynchronously, typically on a thread pool thread.
  • Key Types:
    • Task: Represents an operation without a return value.
    • Task<T>: Represents an operation that returns a value of type T.
  • Creation:
    • Task.Run(): Runs a delegate or lambda on a thread pool thread (e.g., Task.Run(() => DoWork())).
    • Task.Factory.StartNew(): More customizable but less common.
    • new Task(): Creates a task but requires manual starting with .Start().
  • Awaiting: Use await with async methods to wait for task completion without blocking (e.g., await Task.Delay(1000)).
  • Status: Tasks have states like Running, Completed, Faulted, or Canceled.
  • Cancellation: Use CancellationToken to cancel tasks gracefully.
  • Exception Handling: Use try-catch around await or check Task.Exception for errors.
  • Common Use Cases: I/O-bound operations (e.g., file/network access), CPU-bound work, parallel processing.

Example:

async Task DoWorkAsync()
{
    await Task.Run(() => Console.WriteLine("Running..."));
}
