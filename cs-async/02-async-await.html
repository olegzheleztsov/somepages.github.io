In C#, async and await are keywords used to simplify asynchronous programming, allowing non-blocking execution of code, especially for I/O-bound or long-running operations.

  • async Modifier:

    • Marks a method as asynchronous, allowing it to use await.
    • Must return Task, Task<T>, or ValueTask<T> (or void in rare cases, e.g., event handlers).
    • Indicates the method can perform asynchronous operations.

  • await Keyword:

    • Used inside async methods to pause execution until a Task completes, without blocking the calling thread.
    • Returns control to the caller (e.g., UI thread) while the task runs, resuming when the task finishes.
    • Unwraps the result of a Task<T> (e.g., int result = await GetNumberAsync()).

  • How It Works:

    • async methods run synchronously until the first await is encountered.
    • await schedules the rest of the method to resume after the task completes, using a state machine under the hood.
    • Enables responsive apps (e.g., UI stays active during network calls).

  • Key Points:

    • Use for I/O-bound tasks (e.g., HTTP requests, file operations) or CPU-bound tasks (with Task.Run).
    • Exceptions are captured and propagated through the Task, caught with try-catch around await.
    • Avoid async void except for event handlers; prefer async Task.
    • Use ConfigureAwait(false) for library code if context (e.g., UI thread) isnâ€™t needed.

  • Example:

public async Task<string> GetDataAsync()
{
    HttpClient client = new HttpClient();
    string result = await client.GetStringAsync("https://api.example.com");
    return result;
}
  • Benefits:
    • Cleaner code than callbacks or manual thread management.
    • Improves scalability by freeing threads during I/O waits.
    • Maintains responsiveness in UI applications.
