In C#, an AggregateException is a special exception type used to encapsulate multiple exceptions that occur during concurrent or asynchronous operations, such as those involving Tasks. Itâ€™s part of the Task Parallel Library (TPL) to handle scenarios where multiple errors need to be reported together.

  • Purpose: Groups multiple exceptions into a single exception object, making it easier to handle errors from parallel or async tasks.

  • Common Scenarios:

    • Thrown by Task or Task<T> when one or more operations fail (e.g., in Task.WhenAll).
    • Occurs in parallel operations like Parallel.For or Parallel.ForEach.

  • Key Properties:

    • InnerExceptions: A read-only collection of exceptions that caused the AggregateException.
    • InnerException: Returns the first inner exception (less commonly used).

  • Handling:

    • Use try-catch to catch AggregateException.
    • Use Flatten() to simplify nested AggregateExceptions into a single level.
    • Use Handle(Func<Exception, bool>) to process each inner exception individually; return true for handled exceptions, false to rethrow unhandled ones.

  • Example:

try
{
    Task[] tasks = new[]
    {
        Task.Run(() => throw new InvalidOperationException("Error 1")),
        Task.Run(() => throw new NullReferenceException("Error 2"))
    };
    Task.WhenAll(tasks).Wait();
}
catch (AggregateException ae)
{
    ae.Flatten().Handle(ex =>
    {
        Console.WriteLine(ex.Message); // Outputs: Error 1, Error 2
        return true; // Mark as handled
    });
}
  • Async/Await Context:
    • With await, the first inner exception is typically thrown instead of the AggregateException.
    • Example:
try
{
    await Task.WhenAll(tasks);
}
catch (Exception ex) // Catches first inner exception (e.g., InvalidOperationException)
{
    Console.WriteLine(ex.Message);
}
  • Key Points:
    • Simplifies error handling for concurrent tasks.
    • Always check InnerExceptions for full error details.
    • Flatten and Handle are useful for complex scenarios with nested exceptions.
    • In async/await, you often deal with the first inner exception directly.
