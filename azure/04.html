Availability Zone (AZ) ‚Äì Official Definition

An An Availability Zone is a physically separate, isolated fault domain inside a single Azure Region.
Each AZ consists of one or more complete data centers with independent power, cooling, networking, and UPS/generators.

Microsoft guarantees:

  • At least 3 Availability Zones in every region that supports them (most modern regions do).
  • Zones are separated by enough distance to avoid shared fate (fire, flood, power grid failure), but close enough for <2 ms latency between them.
