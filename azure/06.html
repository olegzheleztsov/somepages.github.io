The Correct & Modern Way: Azure Role-Based Access Control (RBAC)

Azure uses Azure RBAC (Role-Based Access Control) ‚Äì not old classic admins or AAD groups alone.

You assign roles to users/groups/service principals at a specific scope (subscription, resource group, or individual resource).

Step-by-Step: How to Grant Access (You Will Be Asked This!)

In Azure Portal:

  1. Go to the scope (Subscription / Resource Group / specific resource)
  2. Access control (IAM) blade ‚Üí ‚Äú+ Add‚Äù ‚Üí ‚ÄúAdd role assignment‚Äù
  3. Select Role (e.g., Contributor)
  4. Assign access to: User, group, or service principal
  5. Select the member (person or AAD group)
  6. Review + assign

Via Azure CLI (what Tech Leads actually use):

# Give Contributor at resource group level
az role assignment create \
  --assignee john.doe@contoso.com \
  --role "Contributor" \
  --resource-group MyApp-Prod-RG

# Reader access to entire subscription
az role assignment create \
  --assignee auditors@contoso.com \
  --role "Reader" \
  --scope /subscriptions/00000000-0000-0000-0000-000000000000

# Key Vault secrets read for an app (via Managed Identity)
az role assignment create \
  --assignee /subscriptions/.../resourceGroups/MyApp-RG/providers/Microsoft.ManagedIdentity/userAssignedIdentities/myapp-identity \
  --role "Key Vault Secrets User" \
  --scope /subscriptions/.../resourceGroups/Secrets-RG/providers/Microsoft.KeyVault/vaults/prod-kv
